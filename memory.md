# Memory Model

## Semantic Memory

حافظه semantic برای حقایق پایدار است:

- قرارداد API
- قواعد کسب‌وکار
- glossary
- configuration policy

## Episodic Memory

حافظه episodic برای رخدادهای زمان‌مند است:

- تصمیم معماری
- کشف/رفع باگ
- migration
- incident
- deployment
- تلاش ناموفق
- یافته performance/security

## Graph Memory

حافظه graph برای روابط ساختاری است:

- controller به service
- service به entity
- entity به migration
- namespace/module relationships

## رابطه بین سه حافظه

- semantic می‌گوید «چه چیزی درست است».
- episodic می‌گوید «چه اتفاقی افتاد».
- graph می‌گوید «چه چیزهایی به هم وصل‌اند».
