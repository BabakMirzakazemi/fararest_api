مستند فلوی مقداردهی اولیه مجوزها

این نسخه رفتار جدید مقداردهی اولیه مجوزها را در زمان startup توضیح می‌دهد تا وجود permission تکراری باعث خطای EF Core و توقف اجرای برنامه نشود.

نسخه v3 تاریخ 2026-05-14 Scope Auth / Authorization Seed

1) نمای کلی رفتار جدید

Program startup -> InitializeDatabase -> AuthorizationDataInitializer -> load catalog permissions -> ignore duplicate permission keys in memory -> upsert tracked permissions without forcing Update on Added entities -> save permissions -> map role permissions -> map plan permissions -> ensure seed user plan

APIهای درگیر: بدون endpoint جدید؛ این رفتار در startup برنامه قبل از سرویس‌دهی API اجرا می‌شود.

جدول‌های درگیر: accounts_permission، accounts_role_permission، accounts_plan_permission، accounts_user_plan_subscription، license_plan

2) جزئیات تغییر در seed مجوزها

در نسخه جدید، catalog مجوزها قبل از هرگونه upsert از نظر permission key یکتا می‌شود. اگر یک کلید تکراری در تعریف‌های حافظه وجود داشته باشد، فقط اولین تعریف پردازش می‌شود و startup ادامه پیدا می‌کند.

همچنین وقتی یک permission یا plan از دیتابیس با query tracked خوانده شده باشد، تغییرات آن فقط روی همان entity اعمال می‌شود و دیگر Repository.Update روی آن صدا زده نمی‌شود. این کار از تغییر state اشتباه برای entityهای تازه‌ساخته‌شده با کلید موقت جلوگیری می‌کند.

APIهای درگیر: ندارد

جدول‌های درگیر: accounts_permission، license_plan
