Program Flow Document v5

This version documents the agent-safe tooling endpoints added for MCP-friendly integration.

Impacted APIs

API Purpose Impacted Tables
GET /api/admin/v1/AgentTools/CapabilitiesAsync Returns the recommended agent tool catalog and MCP-safe capability map. None
GET /api/admin/v1/AgentTools/DatabaseStatusAsync Returns provider, connectivity, and migration status without direct SQL access. __EFMigrationsHistory via EF metadata APIs
GET /api/admin/v1/AgentTools/MemoryStatusAsync Returns episodic memory freshness and coverage for agent decisions. memory_episode

End-to-End Behavior

Agents should prefer application-level APIs over direct database access. The new flow is:

  1. Inspect capabilities from AgentTools/CapabilitiesAsync.
  2. Check DatabaseStatusAsync when migration or environment state matters.
  3. Check MemoryStatusAsync to determine whether episodic memory is populated and recent.
  4. Use episodic memory APIs for search and record workflows.

Design Rule

Direct PostgreSQL MCP access is intentionally not part of the current flow. The project favors read-only, boundary-safe adapters first.
