Program Flow Document v6

This version documents lightweight hybrid retrieval, retrieval-quality evaluation, and stronger episodic-memory workflow guidance.

Impacted APIs

API Purpose Impacted Tables
POST /api/admin/v1/Episodes/SearchAsync Supports lightweight hybrid ranking using text, tags, references, importance, and recency. memory_episode, memory_episode_tag, memory_episode_reference
POST /api/admin/v1/Episodes/EvaluateSearchAsync Evaluates retrieval quality for a known query using recall, precision, and reciprocal rank. memory_episode, memory_episode_tag, memory_episode_reference
GET /api/admin/v1/AgentTools/MemoryStatusAsync Returns memory freshness plus workflow recommendations for when to search and record. memory_episode

End-to-End Behavior

  1. Agent checks memory/database status for sensitive work.
  2. Agent runs hybrid-ranked search before migrations, auth/security edits, and repeated fixes.
  3. Agent records durable outcomes with better structured context and stable references.
  4. Team evaluates repeated query patterns with EvaluateSearchAsync to tune retrieval quality.

Design Rule

The retrieval upgrade remains relational and provider-portable. No vector store, RAG stack, or direct database MCP access was added in this phase.
