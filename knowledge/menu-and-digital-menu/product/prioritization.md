# Product Prioritization Guide - Fararest Menu Module

## MVP Principle

MVP should not attempt to implement every ideal capability.

Priority should be based on:

1. Operational correctness
2. Sales usability
3. Integration with ERP
4. Scope control
5. Ability to extend later

## Suggested MVP - Menu Management

### Must Have

- category management
- item management
- item type
- base price
- active/inactive status
- unavailable status
- branch/channel activation
- simple modifiers
- basic search/filter
- role-based access
- audit log for sensitive changes

### Should Have

- bulk edit
- import/export
- scheduled availability
- image/description warnings
- recipe connection
- tax policy
- menu preview

### Later

- A/B testing
- AI recommendations
- advanced profitability analysis
- approval workflows
- central menu with branch override
- official invoicing metadata automation

## Suggested MVP - Digital Menu & Kiosk

### Must Have

- public menu by QR/link
- mobile-first RTL UI
- categories and item cards
- item detail
- availability display
- simple cart
- dine-in/takeaway selection
- order submission to backend/POS
- basic branding
- preview before publish

### Should Have

- kiosk touch mode
- online payment
- coupons
- scheduled menu
- item suggestions
- receipt/pre-invoice
- analytics events

### Later

- Digital Signage playlist
- adaptive menu by time/weather/crowd
- smart recommendation by margin/stock
- loyalty personalization
- A/B testing
- multilingual managed translations

## Do Not Put in Early MVP Unless Required

- full AI recommendation engine
- advanced A/B testing
- complex multi-brand orchestration
- deep CRM personalization
- full Digital Signage automation
- dynamic weather-aware menu
